📊 命令對照表
命令 群組需滑動 處理方式 回應 可能的錯誤
start ❌ 否 直接發送工具選單 分開的多個工具訊息 無
stop ❌ 否 停止所有運行中工具 「已停止工具：xxx」 無運行中工具時提示
滑動工具訊息 ✅ 是 直接啟動對應工具 「已啟動 xxx」 訊息 ID 未找到
天氣查詢 ❌ 否 智能指令優先處理 完整天氣資訊 API 失敗、城市不存在
狀態 ✅ 是 顯示運行中工具列表 工具列表或「無執行中功能」 無
選單 ✅ 是 重新發送工具選單 分開的多個工具訊息 無
其他文字 ✅ 是（需啟動工具） 傳遞給活動工具處理 根據工具邏輯 無活動工具時被忽略
⚠️ 常見錯誤與處理
錯誤 1: 群組中指令被忽略
症狀： 輸入指令後機器人無反應

原因：
• 未滑動回覆機器人訊息
• 不是 start/stop/天氣查詢指令

解決：
• 先輸入 "start" 開啟選單
• 或滑動回覆機器人訊息後輸入指令
• 或使用天氣查詢（無需滑動）
錯誤 2: 工具啟動失敗
症狀： 「啟動 xxx 失敗」

原因：
• 工具索引不存在
• 工具實例化失敗
• 權限問題

解決：
• 檢查工具名稱是否正確
• 查看終端機日誌獲取詳細錯誤
• 重新啟動機器人
錯誤 3: 天氣查詢失敗
症狀： 「查詢天氣失敗」

原因：
• 網路連接問題
• wttr.in API 不可用
• 城市名稱不正確

解決：
• 檢查網路連接
• 使用標準城市名稱（香港、台北、東京等）
• 稍後重試
